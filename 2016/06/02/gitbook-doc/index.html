安装和起步

目录结构

在运行gitbook init的时候会自动生成几个文件。所有在SUMMARY.md里面列出的 Markdown/Asciidoc 文件都会转换成 HTML 文件(以供相应的页面渲染)。支持多语言的书的目录结构稍微有点不同。其实就是在根目录下加一个LANGS.md，把不同语言的版本放到不同的文件夹下面。

文件 作用

README.md 书的封面介绍(required)
SUMMARY.md 目录(optional)
GLOSSARY.md 术语表(optional)

静态文件和图片

静态文件指的是那些没有列在SUMMARY.md里面的文件。除非是列在忽略文件里面，否则静态文件直接拷贝到输出(也就是 md 文件，不会渲染成 HTML)。

忽略文件和文件夹

Gitbook 会读取.gitignore,.bookignore和.ignore文件来获得应该要忽略的文件。格式就是常规的.gitignore的格式。

结构

默认情况下，根目录下的诸如README.md和SUMMARY.md这些文件都有着其对应的目的。不过，也可以手动配置别的名字：

变量 描述
structure.readme 介绍，默认是README.md
structure.summary 目录，默认是SUMMARY.md
structure.glossary 术语表，默认是GLOSSARY.md
structure.languages 语言，默认是LANGS.md

PDF 选项

变量 描述
pdf.pageNumbers 每一页页脚加上页数(默认 true)
pdf.fontSize 字体大小，默认是 12
pdf.fontFamily 字体，默认是Arial
pdf.paperSize 纸张大小，默认是 a4
pdf.margin.top 上外边距，默认56
pdf.margin.bottom 下外边距，默认56
pdf.margin.right 右外边距，默认62
pdf.margin.left 左外边距，默认62

把内容托管在 Github 上

如果希望别人也可以参与修改和编写，那么把内容托管在 Github 上显然是最好的选择。成功地和 Github repo 建立起联系之后，Github repo 的修改都会触发书的改变。

集成到 Github 需要三步：

  • 建立 Permission
  • 把书链接到一个 Github 仓库
  • 设置 webhooks

  1. 建立 Permission
    要把 Github 集成进来，首先要授权 Gitbook 访问你的 Github账号。你需要在 account settings 里面完成设置。

  2. 把书连接到一个 Github repo
    在你的书的设置页面，填上要连接的 Github repo 的信息：<github-username>/<github-repository-name>。当然了，在此之前你需要新建一个仓库。

  3. 设置 webhooks
    最后一步要对你的仓库进行配置，然后你的仓库一旦更新了，Gitbook 就会得到通知，进而修改书的内容。具体的操作就是在仓库的设置里面找到Webhooks&services，选择 addhooks，把书对应的 URL 填进去。这样一来，一旦仓库有了 Push 事件，Gitbook 就会得到通知。完了之后，可以去 check webhooks 看是不是设置成功了。

设置成功了之后，我们可以在 Github 仓库管理内容，每次仓库有了新的 push，书就会有对应的更新。
